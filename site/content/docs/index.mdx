# Build from schema, not handwritten SQL glue

`comon_orm` is a schema-first ORM toolkit for Dart. The workflow is intentionally simple: model your data in `schema.prisma`, generate a typed client, and use reviewed migrations when the database matters.

Throughout these docs, `runtime` means normal application database access, while `rollout` means preparing or changing a real database schema.

## What this documentation is optimized for

- understanding the actual supported workflow, not an aspirational Prisma clone story
- getting from an empty schema to a running Dart app quickly
- choosing the right migration strategy for PostgreSQL, SQLite, and Flutter local SQLite
- knowing where runtime guarantees stop and rollout review starts

Read the QuickStart first, then use the section docs as reference. The root README stays intentionally short; this site is where the full workflow, examples, and operational guidance live.

## Core path

1. Write `schema.prisma`.
2. Run `check` and `generate`.
3. Prepare the database through reviewed migrations or explicit local upgrades.
4. Open the generated client against runtime schema metadata.
5. Use the generated delegates in application code.
