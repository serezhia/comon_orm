# Code of Conduct

## Our Standards

We expect contributors to keep communication respectful, technical, and focused on improving the project.

Examples of expected behavior:

- assuming good intent and asking clarifying questions before escalating
- giving actionable, evidence-based feedback
- keeping review discussion focused on code, docs, tests, and release impact
- respecting different levels of experience across the community

Examples of unacceptable behavior:

- harassment, discrimination, or personal attacks
- trolling, intimidation, or sustained disruption
- publishing private information without permission
- abusive or hostile communication in issues, pull requests, or discussions

## Scope

This code of conduct applies in project spaces, including issues, pull requests, code review, discussions, and other community channels related to the repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, or other contributions that violate this code of conduct.

Report serious incidents to the repository maintainers through a private channel before escalating them publicly.
